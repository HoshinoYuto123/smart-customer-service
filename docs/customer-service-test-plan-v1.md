## 1. 测试类型与命令

| 类型 | 范围 | 命令/方式 |
|---|---|---|
| 单元测试 | 状态机、策略、脱敏、存储、适配器 | `python -m pytest tests/test_support -q` |
| API/集成 | 认证、所有权、FAQ、自助、队列、工单、评价 | `python -m pytest tests/test_api -q` |
| Agent 集成 | 智能回复、明确人工、高风险、两次未解 | `python -m pytest tests/test_agent -q` |
| 端到端/UI | 用户 MVP、移动布局、坐席工作台 | `node scripts/ui_smoke.js` |
| 权限/异常 | 越权、生产禁 demo 角色、重复提交、未知态 | pytest 专项 |
| 可访问性 | landmark、label、键盘、焦点、aria-live、无横向溢出 | UI smoke + 手动检查 |
| 类型/语法 | Python 类型边界与语法 | `python -m compileall -q app`；mypy 待引入 |
| 代码规范 | 现有仓库无 lint 配置 | `ruff check` 若可用，否则记录未执行 |
| 构建/启动 | FastAPI 导入、静态资源、OpenAPI | startup test + 本地 uvicorn smoke |
